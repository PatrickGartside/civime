<!-- Token auth model (for DATA-03 section) -->
confirm_token (users table):
  Format: 64-character hex string
  Generated: subscription creation time
  Sent: in confirmation email as ?token= URL param
  Validated: GET /subscriptions/confirm looks up in indexed column (idx_users_confirm_token)
  Cleared: not explicitly cleared after use — users.confirmed_email=1 is authoritative state
  Expiry: none — does not expire

manage_token (users table):
  Format: 64-character hex string
  Generated: subscription creation, returned in 201 response body
  Used: ?token= param for all /subscriptions/{id} self-service endpoints
  Scope: one per user, covers all their subscriptions
  Lifecycle: permanent — does not expire or rotate
  Validation: timing-safe comparison against users.manage_token

reminders.confirm_token:
  Format: 64-character hex string
  Generated: POST /reminders time
  Sent: in reminder confirmation email
  Lifecycle: tied to the reminder row; deleted when reminder deleted
  Different from: users.confirm_token (separate column, separate table)
