## Out-of-Scope Issues Discovered

### 1. Broken anchors in PLUGINS.md cross-links to ENDPOINTS.md

**Discovered during:** Plan 06-01, Task 1 (mkdocs build --strict)
**Files:** `docs/plugins/PLUGINS.md`
**Issue:** Three cross-links point to non-existent anchors in `docs/api/ENDPOINTS.md`:
- `../api/ENDPOINTS.md#meetings-1`
- `../api/ENDPOINTS.md#reminders-1`
- `../api/ENDPOINTS.md#councils-1`

**Impact:** MkDocs logs INFO warnings; build exits 0 so not blocking. In a strict environment with `mkdocs.yml` `validation: links: anchors: warn` set to error, this would fail.
**Root cause:** Phase 4 PLUGINS.md was written with anchor IDs that don't match the actual heading slugs generated by MkDocs from ENDPOINTS.md headings.
**Suggested fix:** Update the three links in PLUGINS.md to use the correct anchor slugs, or update the headings in ENDPOINTS.md to match.
**Origin phase:** Phase 04 (wordpress-plugin-documentation)
